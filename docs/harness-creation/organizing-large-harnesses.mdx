# Organizing Large Harnesses

As a harness grows, listing every skill and reference directly in `HARNESS.md` becomes unwieldy. Subdirectories let you group related skills and references together, keeping `HARNESS.md` short while still giving the agent a clear map of what's available.

## SKILLS.md and REFERENCES.md

Each subdirectory should contain a summary file вЂ” `SKILLS.md` for skill directories, `REFERENCES.md` for reference directories. These files describe the contents of the subdirectory in one line per item, allowing `HARNESS.md` to point to the summary rather than listing every individual file.

**`skills/data-access/SKILLS.md`**
```markdown
---
description: Skills for retrieving data from databases and files.
---

- `query-database` вЂ” run a read-only SQL query against the product database
- `read-spreadsheet` вЂ” read rows and columns from a CSV or Excel file
```

**`references/data-sources/REFERENCES.md`**
```markdown
---
description: Reference material about the structure and contents of available data sources.
---

- `schema-overview.md` вЂ” high-level diagram and description of the database schema
- `table-relationships.md` вЂ” foreign key relationships between tables
- `field-definitions.md` вЂ” detailed field-level documentation
- `known-data-quirks.md` вЂ” known data quality issues and workarounds
```

With this in place, `HARNESS.md` stays short:

```markdown
**Skills**
- `skills/data-access/` вЂ” skills for retrieving data (see `SKILLS.md`)
- `skills/reporting/` вЂ” skills for presenting findings (see `SKILLS.md`)

**References**
- `references/data-sources/` вЂ” data structure and quality notes (see `REFERENCES.md`)
- `references/report-standards/` вЂ” formatting and chart conventions (see `REFERENCES.md`)
```
